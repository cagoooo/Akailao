## 📖 專案簡介

「剛好學」是一款專為中小學教師設計的**免費開源課堂互動工具**。  
老師只需準備一個 Firebase 帳號，即可免費建立屬於自己的互動課堂系統，讓學生透過手機或電腦即時參與教學活動！

### ✨ 核心特色

| 特色 | 說明 |
|------|------|
| 🆓 完全免費 | 使用 Google Firebase 免費方案，零主機成本 |
| 📱 無需安裝 | 學生只需打開瀏覽器掃 QR Code 即可加入課堂 |
| 🔒 資料自主 | 所有課堂數據存在您自己的 Firebase，老師完全掌控 |
| 🤖 AI 輔助 | 內建 Gemini AI 自動生成 PIRLS 閱讀理解題組 |
| 📷 **[v3.6.0 新] 拍照/截圖 AI 出題** | 教師直接上傳課本頁面、講義截圖，AI 自動辨識文字並產生完整 PIRLS 四層次題組 |
| 🎛️ **[v3.7.0 新] 是非/選擇題雙模式切換** | 點擊後先選「立刻作答」或「備題作答」，備題模式支援文字 + 圖片 + Gemini 辨識，學生端同步看到題目再答 |
| 🎮 **[v3.7.15 新] 班級即時排行榜** | 學生等待時可玩記憶配對遊戲，完成後即時看到自己排名 + 班上同學完成通知 + 可再次挑戰 |
| 📊 即時監控 | 即時顯示學生作答進度、注意力監控、排行榜 |

## 🚀 正式部署（使用自己的 Firebase）

### 第一步：建立 Firebase 專案

1. 前往 [Firebase Console](https://console.firebase.google.com/)
2. 點擊「**新增專案**」，依指示建立專案（可停用 Google Analytics）
3. 在左側「**建構**」中啟用以下服務：

**Authentication（身份驗證）：**
- 進入 Authentication → 登入方式 → 啟用「**匿名**」

**Firestore Database（資料庫）：**
- 建立資料庫 → 選「**Standard 版**」→ 位置選「`asia-east1 (Taiwan)`」→ 以「**測試模式**」啟動

### 第二步：設定 Firestore 安全性規則

> ⚠️ **此步驟非常重要！** 測試模式預設 30 天後會拒絕所有請求。

進入 Firestore Database → 點擊「**規則**」分頁，將內容替換為：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

設定完成後點擊「**發佈**」。

### 第三步：取得 Firebase 配置金鑰

1. Firebase Console 左上角齒輪圖示 → 「**專案設定**」
2. 「您的應用程式」→ 「**新增應用程式**」→ 選網頁（`</>`）
3. 為應用程式命名後，複製顯示的 `firebaseConfig` 設定碼

### 第四步：使用配置生成器產生您的專屬 index.html

1. 在瀏覽器開啟本 Repo 中的 **`set.html`**
2. 貼上剛才複製的 Firebase 配置碼
3. 點擊「**生成並下載 index.html**」
4. 下載到的 `index.html` 已嵌入您的 Firebase 設定

### 第五步：部署到 GitHub Pages

1. 在 GitHub 建立一個新的 Repository（或使用現有的）
2. 將下載的 `index.html` 上傳到 Repository
3. 進入 Repository Settings → Pages → 選擇 `main` branch → 儲存
4. 幾分鐘後，您的課堂工具將在以下網址上線：
   ```
   https://您的帳號.github.io/Repo名稱/
   ```

> 💡 **更簡單的方式**：直接使用 [EZPage 發布工具](https://gg90052.github.io/ezpage/) 上傳 index.html，可一鍵發布！

## 🎮 功能介紹

### 👨‍🏫 教師端

進入系統後，輸入密碼登入教師模式，即可建立課堂並選擇互動模式：

#### 九大互動模式

| 模式 | 說明 |
|------|------|
| 🚨 **快速搶答** | 學生按下按鈕搶答，可看到搶答排名 |
| 🎨 **繪圖板** | 學生在畫布上作答，教師可放大查看每位學生的作品 |
| ✅ **是非題** | 學生投票 ✓ 或 ✗，即時顯示統計圖表 |
| 📝 **問答題** | 開放式文字作答，教師可在監控面板看所有回答 |
| 🔢 **選擇題** | 支援多題多選項，自動計分，CSV 匯出 |
| 👥 **互評投票** | 學生提交作品後互相投票評分 |
| 📚 **閱讀測驗** | 全功能閱讀理解，含 AI 生題、計時、螢光筆、自動計分 |
| 🔀 **排序題** | 拖拉排序，支援拖曳動畫與 SVG 連線 |
| 🔗 **配對題** | 左右配對連線，動畫化 SVG 連接線 |

#### 教師監控儀表板功能

- 📊 作答進度追蹤（已提交 / 未提交人數）
- 👀 學生注意力監控（分心警告，靜音模式）
- 🏆 即時排行榜（含加分、扣分功能）
- 🎰 學生抽籤（隨機選取同學回答）
- 📤 一鍵匯出 CSV 成績單
- ♻️ 清除所有作答，重新開始

### 👨‍🎓 學生端

1. 以手機或電腦瀏覽器開啟課堂網址
2. 掃描 QR Code 或輸入教室代碼加入課堂
3. 輸入姓名後即可開始作答
4. 作答後永久鎖定，不得再修改

## 🤖 AI 功能設定（選用）

想使用 AI 自動生成閱讀測驗題組？

1. 前往 [Google AI Studio](https://aistudio.google.com/) 取得免費 API Key
2. 進入系統後，教師模式設定 → 輸入 Gemini API Key
3. 在閱讀測驗模式中，三種出題方式任選：
   - 📝 **貼上文字**：將文章貼入文本框 → AI 生成題組
   - 💭 **描述主題**：在出題要求欄輸入主題描述 → AI 生成文本 + 題組
   - 📷 **[v3.6.0 新] 上傳圖片/截圖**：拍攝或截圖課本、講義、PDF 頁面（支援多張、可 Ctrl+V 貼上剪貼簿） → Gemini Vision 自動辨識文字並生成 PIRLS 題組

> 使用 `gemini-2.5-flash` 多模態模型，支援圖文混合輸入。免費方案每分鐘有 quota 限制。

## ⚙️ 進階設定

### 教師密碼設定

首次進入教師模式時，系統會要求設定密碼。密碼儲存於 Firebase Firestore，修改密碼需清除 Firestore 中的相關文件或聯絡管理員。

### 自訂課堂代碼

教師建立課堂時可自訂 3 碼課堂代碼（英數字），學生使用此代碼加入。

### 雲端同步設定

系統支援多裝置同步（教師主控台 + 學生裝置），所有資料即時同步至 Firebase Firestore。

```

  ├── TailwindCSS    ← 樣式（CDN）
```

## 🔧 常見問題

**Q：學生無法加入課堂？**  
A：確認 Firestore 安全性規則已正確設定，且資料庫狀態正常（非暫停）。

**Q：教師忘記密碼怎麼辦？**  
A：進入 Firebase Console → Firestore Database → 找到 classroom 集合 → 刪除對應的教師設定文件，重新設定。

**Q：AI 生題功能顯示錯誤？**  
A：確認 API Key 有效，且未超出 Gemini 免費方案的請求限制（每分鐘 60 次）。

**Q：Firestore 費用怎麼計算？**  
A：一般課堂使用（每天 10-30 人）完全在 Firebase 免費方案之內，不會產生費用。只有超大規模使用（每天數千次讀寫）才需升級計費方案。
