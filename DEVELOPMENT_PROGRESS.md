#### 🟡 D-1. index.html 拆分為 ES Module ⏱️ 1-2 週｜🎯 中｜🔧 高
- **現況**：22,000+ 行單一 HTML，IDE 載入吃力、版本控制 diff 大

#### 🟡 D-3. Firestore 離線快取 ⏱️ 1 天｜🎯 中

- **建議**：啟用 `enableIndexedDbPersistence()`
- **影響**：學生短暫斷網時仍能繼續作答；節省 Firestore 讀取配額

#### 🟢 D-4. 自動化測試（Playwright E2E）⏱️ 3-5 天｜🎯 中
- **現況**：每次改動都要手動 Chrome MCP 驗證
- **建議**：建立 `tests/e2e/` 跑常見流程
    - 教師建教室 → 學生加入 → 各模式互動
    - 閱讀測驗 AI 出題 → 學生作答 → 教師看結果
- **CI 整合**：commit 觸發 GitHub Actions 跑 E2E

#### ✅ D-5. 升級 GitHub Actions 至 Node 24 — **V3.8.5 完成**
- **現況**：每次部署都跳警告 "Node.js 20 deprecated by 2026-09-16"
- **建議**：在 `.github/workflows/deploy.yml` 加 `env: FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true`
- **影響**：消除 GitHub Actions 警告

### 🎯 階段 A 收尾：V3.8.6 ~ V3.8.9（短期完善）

#### 🟡 A-6. AI 個人化學習評語生成（基於 V3.8.5 的 perStudentScores）⏱️ 1-2 天 ｜🎯 高
- **依賴**：V3.8.5 已歸檔每位學生個別答對率
- **流程**：
    1. 在「👤 個人成績」Tab 新增「✨ AI 評語」按鈕
    2. 把該學生的歷史成績 + 平均 + 進步幅度餵給 Gemini
    3. 產出「鼓勵 + 建議改進方向 + 家長溝通用語」三段式評語
- **Prompt 範例**：
    ```
    學生「王小明」最近 8 次測驗的成績與班級平均對照：
    [日期 / 模式 / 個人答對率 / 班級平均答對率 / 排名]
    
    請以親切溫暖、避免比較傷害的語氣，產出：
    1. 對學生本人的鼓勵（口語、3-4 句）
    2. 給家長的客觀觀察（書面、具體事實）
    3. 建議的改進方向（針對性、可執行）
    ```
- **效益**：老師批改評語時間減 80%，且品質更穩定

#### 🟢 A-7. 即時加題完整版：編輯既有題目 + 凍結未交者 ⏱️ 3-5 天 ｜🎯 中｜🔧 高

- **凍結未交者**：寫入 `frozen: true` 欄位，學生端 listener 偵測 → 鎖介面 → 顯示「老師正在更新題目，請稍候」
- **重新計分**：題目改後一鍵重算所有人成績（沿用 V3.7.x 的 rescoreAllStudents 邏輯）

#### 🟢 A-8. AI 評析：「這題太難 / 太簡單」自動建議 ⏱️ 1 天 ｜🎯 中

#### ✅ A-9. 教師專屬「題目品質統計」— **V3.8.8 完成**

### 🐛 V3.8.9 (2026-04-16) - HOTFIX：V3.8.7 Tailwind 本地化造成所有 modal 預設不隱藏

#### 🔥 嚴重 Bug：重新整理頁面後黑色蒙版蓋滿全螢幕
**現象**：使用者重整頁面後，「快速投票統計」+ ZoomableImageViewer 兩個 modal 同時跳出，黑色蒙版蓋滿全頁面，無法操作。

#### 🔍 根本原因（CSS source order 衝突）
V3.8.7 把 Tailwind 從 CDN 改為本地 build，但載入順序變了：
```html
<head>
  <link rel="stylesheet" href="tailwind-build.css">  <!-- (1) -->
  <style>                                             <!-- (2) -->
    .magnified-image-modal { display: flex }
    .zoomable-image-viewer { display: flex }
    ...
  </style>
</head>
```
- `tailwind-build.css` 中：`.hidden { display: none }`（無 !important）
- inline `<style>` 中：`.magnified-image-modal { display: flex }`（無 !important）
- 兩者 specificity 都是 `(0,1,0)`，**source order 後者勝** → modal 預設帶 `class="magnified-image-modal hidden"` 仍被當成 `display: flex` 顯示

#### 💡 為什麼 V3.8.7 之前沒事？
V3.8.7 之前用 `cdn.tailwindcss.com` 是 v3 風格 JIT，會在 runtime 動態注入 CSS，可能用 `@layer utilities` 包裝給予 utilities 較高優先級，正好覆寫 inline style。

#### ✅ 修復方案：用 `:not(.hidden)` 修飾 selector
```css
.magnified-image-modal:not(.hidden) {  /* ← 加上 :not(.hidden) */
    display: flex;
    /* ... */
}
.zoomable-image-viewer:not(.hidden) {
    display: flex;
    /* ... */
}
```
**為何不用 `!important`？** 會破壞 Tailwind responsive utilities，例如 `class="hidden sm:inline"`（手機隱藏、平板以上顯示）會永遠 hidden。

#### 📊 影響範圍
- 受影響：30+ 個帶 `.magnified-image-modal hidden` class 的 modal（投票統計、查看題目、編輯答案、即時加題、結束互動確認等等）
- 受影響：ZoomableImageViewer（學生端題目圖片放大檢視器）
- 修復後：所有 modal 預設正確隱藏，僅在程式呼叫 `classList.remove('hidden')` 時才顯示

#### 📌 經驗教訓
換 CSS framework 載入方式時，務必檢查與 inline `<style>` 的 specificity 互動，特別是 `display:` 屬性。

#### 1. 🤖 AI-1：AI 自動批改問答題 ⭐⭐⭐ ⏱️ 2-3 天 🔧 中
- **現狀**：問答題（文字題）只能人工閱卷，教師批改 30 人要花 20 分鐘

  - 名單存 Firestore `classrooms/{code}/roster/{studentId}`
  - 學生進入時偵測名單 → 顯示下拉選單取代文字輸入
  - CSV 匯出時依座號排序

- **前置依賴**：無

#### 3. 🌙 UX-5：深色模式 ⭐⭐ ⏱️ 1 天 🔧 低
