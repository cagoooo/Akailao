## 🔮 未來優化建議（詳細版）

> 按照實作難度與影響力分為三個層次：🟢 低難度、🟡 中難度、🔴 高難度

---

### 📱 UX / 使用者體驗改善

#### 🟢 低難度（可快速實作）

**1. 學生端作答進度提示**
- 目前學生不知道全班有多少人作答
- 建議：在學生端底部加一條進度條「班級作答進度 12/30」
- 給學生正向壓力，促進提交速度

**2. 閱讀測驗送出後的動畫慶祝效果**
- 目前送出後只顯示文字確認
- 建議：加入 confetti 動畫 + 顯示「你是第 N 位完成的同學！」
- 大幅提升學生提交意願

**3. 教師端「快速評語」按鈕**
- 在學生卡片旁加幾個預設評語按鈕（💪 Very good！、📝 Try again）
- 點擊後學生端即時顯示老師的評語彈窗
- 增強師生互動感

**4. 倒數計時聲音提示**
- 閱讀測驗最後 30 秒時，送出按鈕輕微震動（CSS animation）
- 讓學生注意計時將結束

**5. 深色模式（Dark Mode）**
- 學生長時間閱讀文章，深色模式可減少眼睛疲勞
- 用 CSS 變數實作，加一個切換按鈕

---

#### 🟡 中難度（需要一至三天）

**6. 閱讀測驗「逐步揭露題目」模式**
- 目前所有題目一次全顯示
- 建議：支援老師設定「每 N 分鐘解鎖下一題」的逐步模式
- 強迫學生按閱讀節奏完成

**7. 學生端個人成績歷史記錄**
- 學生可以查看自己歷次測驗的分數趨勢
- 資料存 localStorage + Firestore（以姓名為 key）
- 顯示簡單折線圖（D3.js 已引入）

**8. 教師端「自動暫停回覆」計時器**
- 老師設定幾分鐘後自動暫停回覆（不需手動按按鈕）
- 讓教師可以更專注於巡視學生

**9. 配對題 / 排序題支援圖片**
- 目前只支援文字選項
- 建議：選項可以是圖片 URL（或 base64）
- 適合低年級或視覺化學科

**10. 分組功能（Group Mode）**
- 學生加入時可選擇組別（A/B/C/D 組）
- 教師可查看各組的作答統計
- 適合分組競賽或小組討論

**11. 離線作答支援（Service Worker 增強）**
- 學生進入測驗後，即使網路中斷也能繼續作答
- 完成後重新連線自動上傳
- 解決網路不穩定教室的痛點

**12. 學生端浮動字典工具**
- 閱讀測驗時，長按中文詞彙可查詞義
- 串接免費中文字典 API
- 適合閱讀理解訓練

---

#### 🔴 高難度（需要一週以上）

**13. Firebase Functions 後端加密**
- 目前所有邏輯在前端，有心人可繞過送分限制
- 建議：將計分邏輯移至 Firebase Functions（Cloud Run）
- 解決答案洩漏、作弊等安全問題

**14. 多班級管理後台**
- 目前一個 Firebase 帳號只支援一間教室
- 建議：教師可建立多個課堂（如「甲班」「乙班」），切換管理
- 需要 Firestore 資料結構重新設計

**15. 學生帳號系統（非匿名）**
- 目前學生以姓名字串辨識，可能重複或偽冒
- 建議：整合 Google 登入（Firebase Auth Google Provider）
- 用 UID 作為主鍵，支援跨裝置、跨課堂的個人歷史記錄

**16. 題庫管理系統**
- 教師可儲存常用題組（選擇題、閱讀測驗）
- 建立個人題庫，下次一鍵載入
- 資料存 Firestore + localStorage 雙備份

**17. 班級出席記錄**
- 學生掃 QR Code 加入課堂同時自動記錄出席
- 教師可查看今日/歷次課堂的出勤統計
- 匯出出席 CSV

**18. WebRTC 即時視訊功能（進階）**
- 教師可開啟簡易視訊廣播給學生端
- 學生可舉手提問（顯示在教師端）
- 結合現有互動功能形成完整遠距教學解決方案

---

### 🤖 AI 功能擴展

**19. AI 自動批改開放式作答（問答題）**
- 目前問答題只能人工閱卷
- 用 Gemini API 根據標準答案自動評分（0-100 分）
- 顯示 AI 評語，教師可覆蓋修正

**20. AI 生成排序題 / 配對題**
- 目前 AI 只能生成選擇題和閱讀理解題
- 擴展到排序題（邏輯順序）和配對題（詞彙對義）
- 一鍵生成完整單元測驗

**21. AI 個人化學習建議**
- 根據學生歷次測驗表現，AI 生成個人化複習建議
- 「你在 Q3〜Q5 題型較弱，建議加強...」
- 列印成親師溝通的學習報告

**22. AI 難度自動調整（Adaptive Learning）**
- 根據班級整體答對率，AI 自動建議下一道題的難度
- 讓老師省去選題的時間

### ⚡ 效能優化

**23. 虛擬清單渲染（Virtual Scroll）**
- 當班級人數超過 50 人時，卡片渲染效能下降
- 建議：實作 Virtual Scroll，只渲染可視範圍內的卡片

**24. Firestore 讀取快取優化**
- 目前部分資料每次都重新讀取
- 使用 `onSnapshot` cache + local state diff 減少讀取次數
- 在免費方案下支援更大規模班級

**25. 圖片壓縮（繪圖模式）**
- 目前繪圖直接存 base64 到 Firestore，大圖會超出文件大小限制
- 建議：上傳前自動壓縮到 200KB 以下
- 或改用 Firebase Storage（免費 5GB）

---

### 🔒 安全性強化

**26. 閱讀測驗答案防偷看**
- 教師設定題目時，正確答案不應存在前端可讀取的位置
- 建議：正確答案用 Firebase Functions 加密存儲
- 學生提交後才由後端比對計分

**27. 速率限制（Rate Limiting）**
- 目前學生理論上可以無限次提交
- 用 Firestore rules + timestamp 限制每個 session 只能提交一次
- 防止刷分行為

**28. 教師端 IP 白名單（進階）**
- 只允許特定網路（學校 IP）進入教師模式
- 防止外部人員惡意登入管理介面
