### 📱 UX / 使用者體驗改善

#### 🟢 低難度（可快速實作）

**UX-1. 學生進入後「上次姓名」自動帶入**
- 現有 `localStorage.getItem('lastStudentName')` 已儲存姓名
- 建議：在學生姓名輸入框預填上次使用的姓名，學生只需確認後按送出即可
- 搭配 `?classroom=` 連結，三步驟縮短為「開連結 → 確認姓名 → 進入」

**UX-2. 班級分享連結 QR Code 美化**
- 目前 QR Code 為純黑白
- 建議：在 QR Code 中央嵌入學校 logo 或 Akailao 圖標，並加上彩色邊框
- 貼在教室門口 / 印在講義上更美觀，提升掃碼率

**UX-3. 閱讀測驗送出後的慶祝動畫**
- 目前送出後只顯示文字確認
- 建議：加入 confetti 動畫（可用 canvas-confetti.js）+ 顯示「你是第 N 位完成的同學！」
- 大幅提升學生提交意願

**UX-4. 倒數計時視覺警告（最後 30 秒）**
- 閱讀測驗最後 30 秒時，計時器自動變紅色 + 輕微 pulse 動畫
- 最後 10 秒時，送出按鈕也跟著閃爍
- 不需另外聲音提示，對教室環境友善

**UX-5. 深色模式（Dark Mode）**
- 學生長時間閱讀文章，深色模式可減少眼睛疲勞
- 用 CSS 自訂屬性（`--bg`, `--text`）實作，加一個切換按鈕
- 設定存 localStorage，下次自動帶入

**UX-6. 教師端「快速評語」一鍵發送**
- 在學生卡片旁加幾個預設評語按鈕（💪 Very good！、� 再想想、�📝 訂正後重交）
- 點擊後學生端即時顯示老師的評語浮動訊息（Toast notification）
- 不需額外介面，大幅增強師生互動感

**UX-7. 學生端作答人數進度條**
- 在學生等待頁底部加一條進度條「班級作答進度 12/30」
- 給學生正向壓力，促進提交；給未交的學生緊迫感
- 只需監聽 `studentResponses` 的文件數量

### 🤖 AI 功能擴展

**AI-1. AI 自動批改開放式作答（問答題）**
- 目前問答題只能人工閱卷
- 老師設定參考答案 → Gemini 自動給分（0-10）+ 評語
- 教師可覆蓋 AI 分數，AI 只作參考建議
- 搭配 Firebase Functions 呼叫 Gemini API（避免前端暴露 Key）

**AI-2. AI 生成排序題 / 配對題**
- 目前 AI 只能生成選擇題和閱讀理解題
- 輸入課文段落 → Gemini 生成排序題（邏輯步驟順序）
- 輸入詞彙表 → Gemini 生成配對題（詞彙配釋義）
- 一鍵生成完整多模式測驗

**AI-3. AI 個人化學習建議報告（學期末）**
- 根據學生整學期歷次測驗表現，Gemini 生成個人化學習評語
- 「你在閱讀測驗的推論題型較弱，建議加強...」
- 一鍵列印成親師溝通的學習報告（PDF 匯出）

**AI-4. AI 難度自動調整建議（Adaptive Learning）**
- 根據班級整體答對率，AI 自動建議下一道題的難度
- 例如：「本題全班答對率 92%，建議換一道 B2 難度的題」
- 讓老師省去選題的時間，提升教學效率

**AI-5. AI 即時口語作答轉文字**
- 學生可以用麥克風口頭回答（Web Speech API → 轉文字）
- 適合口語訓練、英語課、特殊需求學生
- 轉文字成功後自動填入問答題輸入框

- 目前學生以姓名字串辨識，可能重複或偽冒

- 老師可看到學生的 Google 帳號頭像，確認身份

**ARCH-4. 題庫管理系統**

- 可分享題庫給其他老師（產生分享連結）

**ARCH-5. 班級出席記錄**

- 支援遲到記錄（設定上課時間，N 分鐘後加入計為遲到）
- 一鍵匯出出席 CSV

### ⚡ 效能優化

**PERF-1. 虛擬清單渲染（Virtual Scroll）**
- 當班級人數超過 50 人時，卡片渲染效能明顯下降
- 實作 Virtual Scroll：只渲染可視範圍內的卡片（±5 張緩衝）
- 可用 Intersection Observer API 實作，無需引入外部套件

**PERF-2. 繪圖模式圖片壓縮 + Firebase Storage**
- 目前繪圖直接存 base64 到 Firestore，超過 1MB 會報錯
- 上傳前自動壓縮到 200KB 以下（現有 canvas toDataURL 已有壓縮）
- 改用 Firebase Storage 存圖片，Firestore 只存 download URL（更穩定）

**PERF-3. 程式碼分割（Code Splitting）**
- `index.html` 目前約 984KB（19,000+ 行），所有邏輯混在一起
- 建議：考慮遷移到 Vite + ES Module 架構，按功能懶載入
- 首頁載入時間可從 3-5 秒縮短至 1 秒以內

**PERF-4. Firestore 讀取成本控制**
- 目前部分資料每次進入都重新讀取
- 使用 Firestore 離線快取（`enableIndexedDbPersistence`）
- 在免費方案（50,000 次/日）下支援更大規模使用

### 🔒 安全性強化

**SEC-1. 閱讀測驗答案防偷看（後端計分）**
- 正確答案目前可在前端 JS 中讀取（開 DevTools 即可看到）
- 建議：正確答案存 Firestore 私有路徑（`private/answers`），只有 Firebase Functions 可讀
- 學生提交後由 Functions 後端比對計分，前端看不到正確答案

**SEC-2. 速率限制（防刷分作弊）**
- 目前學生理論上可重複提交
- Firestore Rules 限制：每個 `studentResponse` 文件若 `answer` 欄位已存在，禁止覆寫
- 搭配前端鎖定按鈕，雙重防護

**SEC-3. 分心監控資料隱私保護**
- 目前分心記錄存在 Firestore 公開路徑，學生理論上可自行清除
- 建議：分心資料存入教師私有路徑，學生無寫入/刪除權限
- 更新 Firestore 安全規則

### 🎮 進階互動模式（新模式）

**MODE-1. 投票/問卷調查模式（Survey）**
- 教師發起一道無標準答案的問卷（「你最喜歡哪種上課方式？」）
- 結果即時顯示長條圖/圓餅圖
- 匿名模式：不顯示是誰選了什麼選項

**MODE-2. 分組競賽模式（Team Battle）**
- 學生加入時選擇組別（紅隊/藍隊/綠隊/黃隊）
- 每次互動，得分累計至隊伍而非個人
- 教師端顯示隊伍積分板（大字體，投影給全班看）

**MODE-3. 文字雲即時生成（Word Cloud）**
- 學生輸入一個詞彙，全班詞彙即時合併成文字雲
- 用 D3.js 或 wordcloud2.js 渲染
- 適合腦力激盪、關鍵詞梳理

**MODE-4. 班級相片牆（Photo Wall）**
- 學生用繪圖模式提交作品後，教師一鍵切換至「相片牆」全螢幕展示
- 所有學生作品排成網格，可點擊放大
- 適合美術課創作展示或作業發表

**MODE-5. 課堂問卷（學生對老師的回饋）**
- 課程結束時，老師發起「課後回饋」問卷
- 學生匿名評分（1-5 星）+ 一句話建議
- 教師可查看整學期的教學回饋趨勢
